1. var let const的区别

用法

var 和 let 使用上没有区别，const声明只读的常量，不能改变，并不是值不能改变，而是指向的内存地址不能变

var 在全局范围内生效，在声明前可以获取，可以重复声明，
let 和 const 在块级作用域内生效，在声明前不能获取，不能重复声明

变量提升

var 创建和初始化都提升了，所以在赋值前可以访问到 得到undefined
let 和 const 只在创建过程提升，初始化不提升，所以会有暂时性死区

内存分配

var 会直接在栈内存中预分配内存空间
let 和 const 不会预分配内存空间，并在声明时检查之前是否有重复声明，有的话报错

2. bind call apply的区别

共同点

都是修改函数体的this指向

不同点

bind不会立即调用，而是会返回一个新函数

call和apply会立即调用，但所传参数不同
apply第一个参数是this指向，第二个参数是参数数组
call第一个参数是this指向，第二个及以后参数是挨个列举

性能上call比apply好一些，因为参数是数组的话函数内部也要解构一遍

事件循环

js代码在执行过程中，除了依靠函数调用栈来处理调用顺序时，还依靠任务队列来执行其他的一些代码，整个执行过程，称为事件循环

事件循环是js实现异步的一种方法
事件循环只有一个，但任务队列可以有多个，任务队列又分为宏任务和微任务

执行过程

执行宏任务，然后执行该宏任务产生的微任务，如果微任务在执行过程中产生了新的微任务，则继续执行微任务，微任务执行完毕后，再回到宏任务中进行下一轮循环

async/await执行顺序

await 后面的函数执行完后，会产生一个微任务，这个微任务会直接跳出async函数，执行其他代码，然后再执行
