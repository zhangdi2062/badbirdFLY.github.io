颜色继承

CSS3关键字：currentColor // 当前文字的颜色
例如border会默认取得当前元素的color的颜色值或者继承父级color的色值

svg设置 fill: currentColor可以通过设置color改变图形颜色

兄弟选择器～ +

注意两者都表示选择元素之后的兄弟节点

~表示元素后面所有的兄弟节点，多个

+表示元素后一个相邻的兄弟节点，一个

目前css还没有可选择前一个兄弟节点的方法，可以通过父元素选择，或js的previousElementSibling

:first-child和:first-of-type

问题：当有如下dom结构时，想通过css选择class为cate并且不包括第一个子元素，正常的想法是.cate:not(:first-child)，结果并不能实现，却发现.cate:not(:nth-child(2))是符合的，但显然不符合语意

解决：将:not(:first-child)换成:not(:first-of-type),将class="title"的元素类型换为<p>标签

答疑：:first-child 匹配的是父元素的第一个子元素，可以说是结构上的第一个子元素。

:first-of-type 匹配的是父元素下相同类型子元素中的第一个，比如 p:first-of-type，就是指所有类型为p的子元素中的第一个。这里不再限制是第一个子元素了，只要是该类型元素的第一个就行了。

复习回顾，为啥不用overflow-x: hidden呢？

回答来了：

MDN上说

注意: 设置一个轴为visible（默认值），同时设置另一个轴为不同的值，会导致设置visible的轴的行为会变成auto。

也就是说设置了overflow-x: hidden，会默认overflow-y为 auto，即使设置了visible也不行，
而auto则为超出内容会提供滚动条，不超出隐藏滚动条

另一种解决方式：

overflow-x: clip 但这个兼容性不是很好，刚通过W3C的工作草案，谷歌90+版本支持，是-moz-hidden-unscrollable的替代

隐藏元素的办法 优劣和适用场景

常见三种

opacity: 0、visibility: hidden、display: none

  1. 株连性，父元素设置了opacity和display，是非继承属性，子元素无法改变状态，但visibility可以，父元素隐藏，子元素设置visibility:visible还可以显示出来

  2. display: none 渲染树消失，回流
    visibility: hidden 元素存在，不可点击，重绘
    opacity: 0 元素存在，可点击，重绘

flex 属性

flex是弹性布局，有12个属性，6个作用于父容器，6个作用于子项目

flex 计算题

flex: 0 1 auto是什么意思

项目的属性，不是父级的属性

flex-grow属性定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大。

flex-shrink属性定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。

flex-basis属性定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。

flex属性是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto。后两个属性可选。
该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。

.item {
flex: none | [ <’flex-grow’> <’flex-shrink’>? || <’flex-basis’> ]
}

1. border-image

注意： 一定要先定义边框的样式 border-style 否则渐变无法生效

优点：代码量最少

缺点：可渐变，但不能圆角

示例：

2. 背景渐变

做法：利用伪元素做渐变背景，元素设置和背景同样的颜色，留出边框大小的间距

缺点：如果背景不是纯色而是有透明度的颜色就露馅儿了，而且大背景颜色也不能变

3. clip-path

做法：对背景或img元素都可以设置clip-path，通过多边形polygon设置出边框

缺点：适合做相框，但细边框时圆角也不行

所以确实还没有找到最好的办法，大家有没有更好的办法呢
